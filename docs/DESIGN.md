# 医院患者宣教工作台设计 Brief

## Audience

科室医生、护士、宣教内容维护人员，以及扫码查看宣教内容的患者。

## Primary Workflow

医生选择科室与宣教任务，载入院内已有宣教材料，系统生成患者可读文案、语音宣教和视频预览。医生核对禁忌提醒、检查时间和离院注意事项后发送给患者，患者在手机端查看文字、播放语音和视频内容。

## Information Hierarchy

1. 顶部呈现科室今日任务、已发送人数、患者已读率和异常反馈。
2. 左侧是医生工作台，选择宣教任务并查看本次材料来源。
3. 中央是宣教内容编辑区，突出文案、语音、视频预览和医生确认动作。
4. 右侧是患者手机端预览，确保内容对终端患者可读、可听、可播放。
5. 底部是发送前检查，包括风险提醒、预约信息、医生确认、患者通知和阅读反馈。

## Visual Direction

临床级精密感：浅色医疗工作站 + 克制的深色操作区 + 青绿、琥珀和珊瑚作为状态色。避免科幻大屏、采购材料口径和泛 SaaS 卡片堆，使用连续任务流、患者端预览和医生确认状态建立真实工作台感。

## Density

桌面端采用三栏高密度操作台；移动端保持完整流程但改为纵向堆叠，优先呈现生成链路、发送前核对和患者端预览。

## Constraints

当前界面使用演示数据，不录入真实患者隐私。所有内容必须保留医生确认动作，不能让患者直接接收未经审核的生成内容。
