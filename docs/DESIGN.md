# 医院 AI 宣教生成工作台设计 Brief

## Audience

科室医生、护士、宣教内容维护人员，以及扫码查看宣教内容的患者。

## Primary Workflow

医生选择科室与宣教项目，载入院内已有宣教材料，系统生成患者可读文案、语音宣教和视频片段。医生核对后发送给患者，患者在手机端查看文字、播放语音和宣教片。

## Information Hierarchy

1. 顶部展示科室今日任务、已发送人数、患者已读率和异常反馈。
2. 左侧是医生工作台，选择宣教项目并查看本次材料来源。
3. 中央是宣教内容编辑区，突出文案、语音、视频片段和医生确认动作。
4. 右侧是患者手机端预览，确保内容对终端患者可读、可听、可播放。
5. 底部展示院内工作状态，包括账号、预约信息、资料库、患者通知和发布记录。

## Visual Direction

临床级精密感：浅色医疗工作站 + 深墨色控制条 + 青绿、琥珀和珊瑚作为状态色。避免科幻大屏、采购材料口径和泛 SaaS 卡片堆，使用连续任务流、患者端预览和医生确认状态建立真实工作台感。

## Density

桌面端采用三栏高密度操作台；移动端保持完整流程但改为纵向堆叠，优先展示生成链路和患者端预览。

## Constraints

当前界面使用演示数据，不录入真实患者隐私。所有内容必须保留医生确认动作，不能让患者直接接收未经审核的生成内容。
