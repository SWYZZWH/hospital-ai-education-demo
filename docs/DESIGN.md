# 医院 AI 宣教生成工作台设计 Brief

## Audience

院方科室负责人、信息科、科研项目评审人员，以及需要快速理解项目价值的商务沟通对象。

## Primary Workflow

医生选择试点科室与宣教项目，载入院内已有宣教材料，系统生成患者可读文案、音频宣教、视频脚本与分镜，医生审核后生成患者端展示入口。

## Information Hierarchy

1. 顶部展示真实生成定位、当前服务状态和核心成果。
2. 左侧是医生输入与项目选择，让演示对象明确“从哪开始”。
3. 中央是 AI 宣教生产链路，突出内容生成、音频、视频分镜、审核发布。
4. 右侧是患者端手机预览，证明输出不是内部表单，而是能被患者理解的宣教内容。
5. 底部保留完整项目接口位，表达后续可扩展到院内系统、知识库、公众号、小程序与本地部署。

## Visual Direction

临床级精密感：浅色医疗工作站 + 深墨色控制条 + 青绿、琥珀和珊瑚作为状态色。避免科幻大屏和泛 SaaS 卡片堆，使用连续链路、手机预览和审批状态建立“端到端闭环”的记忆点。

## Density

桌面端采用三栏高密度操作台；移动端保持完整流程但改为纵向堆叠，优先展示生成链路和患者端预览。

## Constraints

本阶段交付可演示的内容生成闭环：模型生成宣教稿、语音文件和 MP4 宣教视频。不接真实 HIS/EMR，不处理真实患者隐私数据，不承诺生产环境私有化联调。
